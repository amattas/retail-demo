# Design Documentation

This section contains the normative technical documentation for the Microsoft
Fabric Retail Demo. It is separated from the task-focused guides so users can
learn, deploy, and present the demo without first navigating implementation
contracts.

| Area | Owns |
| --- | --- |
| [Requirements](requirements/README.md) | Outcomes, constraints, acceptance criteria, stable IDs, and backlogs |
| [Specifications](specifications/README.md) | Exact interfaces, schemas, workflows, states, and errors |
| [Architecture](architecture/overview.md) | Current components, boundaries, dependencies, and data flow |
| [Security](security/threat-model.md) | Threats, controls, assumptions, and residual risks |

Start with the [architecture overview](architecture/overview.md) for a
technical orientation. Use requirements and specifications when implementing
or validating a specific capability.
