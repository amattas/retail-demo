# Dashboards

Real-Time Dashboards for operations and customer experience, backed by the Eventhouse (hot path) with optional Lakehouse Gold overlays. For the Power BI report, see [Semantic Model](./semantic-model.md).

## Important Note

`fabric/dashboards/` contains **two template files**. Dashboards are imported and finalized **manually** in your Fabric workspace (replace the KQL database resource ID after import).

## Templates

### retail-ops.template.json

Operational Real-Time Dashboard with 10 tiles sourced from materialized views and [queryset](./querysets.md) queries:

| Tile | Source |
|------|--------|
| Sales/min by Store (1h) | `mv_store_sales_minute` |
| Top Products (15m) | `mv_top_products_15m` |
| Tender Mix (15m) | `mv_tender_mix_15m` |
| Online Orders (15m) | `online_order_created` |
| Fulfillment Pipeline (24h) | `online_order_*` events |
| Open Stockouts (24h) | `stockout_detected` |
| BLE Presence (30m) | `ble_ping_detected` |
| Zone Dwell Heatmap (30m) | `customer_entered` |
| Truck Dwell by Site (24h) | `truck_arrived`/`truck_departed` |
| Marketing Cost (24h) | `ad_impression` |

### pricing-approval.template.json

Human-in-the-loop **Dynamic Pricing Approval** dashboard (12 tiles) over the pricing approval tables created by `07-pricing-approval-tables.kql` (see [KQL Database](./kql-database.md)). Setup instructions live in `fabric/dashboards/PRICING_APPROVAL_DASHBOARD.md`.

Tiles cover:

- **Pending review**: Pending Recommendations Summary, Recommendations by Department, Top Pending Recommendations by Revenue Impact, Price Change Distribution, Confidence Level Breakdown, Constraint Compliance Status
- **Decisions and audit**: Recent Approvals (24h), Approval Rate by Confidence Level (7d), Realized Revenue Impact (7d, approved only), Approver Activity (7d), Average Decision Time (24h), Top Rejected Recommendations by Reason

The dashboard supports the review → decide → monitor → audit workflow for ML pricing recommendations generated by `14-ml-dynamic-pricing.ipynb`.

**To use a template:**
1. Import the template JSON into your Fabric workspace
2. Replace the KQL database resource ID with your database
3. Update connection credentials
4. Customize visuals and layout

## Data Sources

| Source | Use Case |
|--------|----------|
| KQL DB materialized views | Real-time metrics (hot path) |
| KQL DB event tables | Detail tiles with bounded time ranges |
| Lakehouse Gold tables | Historical overlays and trends |

## Creating Custom Dashboards

1. Navigate to Fabric workspace
2. Create a new **Real-Time Dashboard**
3. Add data sources:
   - KQL database for real-time queries
   - Lakehouse SQL endpoint for historical data
4. Build visuals using materialized views for performance
5. Set auto-refresh interval (e.g., 30 seconds for real-time)

## Best Practices

- **Use materialized views** for dashboard queries (pre-aggregated, fast)
- **Limit time ranges** in KQL queries (e.g., `ago(1h)` for real-time)
- **Cache historical queries** using Lakehouse Gold tables
- **Set appropriate refresh rates** (30s for real-time, 5m for operational)
