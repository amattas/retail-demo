# Datagen to Utility Migration Map

## Success condition and validation loop

The migration is complete when every legacy `datagen` responsibility is either
implemented in `utility`, reworked to a Fabric-native equivalent, replaced by a
Fabric platform capability, or intentionally dropped with a rationale.

Validation loop used for this review:

1. Inventory the legacy implementation now preserved under `datagen-deprecated/`.
2. Inventory the active Fabric-native implementation under `utility/`.
3. Compare table sets, schemas, business rules, streaming events, and operational
   surfaces.
4. Fix mismatches found during review.
5. Run targeted checks for the changed migration paths.

## Intentional non-parity

| Legacy item | Rationale |
|---|---|
| FastAPI dashboard/status/progress/preview surfaces | Replaced by Fabric notebooks, Lakehouse tables, and SQL endpoint access. |
| DuckDB incremental resume/watermarks | Reworked to deterministic full-window overwrite plus run logging. |
| Parquet folder export and Blob upload | Superseded by Lakehouse Delta tables. |
| Event Hubs DLQ/circuit breaker/outbox/Prometheus code | Platform-owned by Fabric Eventstream/Spark/monitoring. |
| Stateful truck queueing/timeouts/recovery | Poor fit for the stateless batch generator; timing and capacity legs remain modelled. |
| Supplier truck leg with `DCID=0` | Supplier inbound is represented as DC inventory transactions. |
| Explicit RTV outbound transactions | Non-restocked returns simply do not re-enter inventory. |

## Bugs and inconsistencies fixed

| Finding | Resolution |
|---|---|
| Docs claimed 19 legacy streaming event types, but the legacy enum has 18. | Corrected docs and added an AST-based event-set parity test. |
| Existing migration notes described restored behavior as still dropped. | Updated audit-facing docs and module comments for online tax, truck capacity, weather, BOGO, and store archetypes. |
| Promotion tests assumed every promotion was `PERCENTAGE`. | Allowed `BOGO` and asserted BOGO promo codes map to `discount_type = BOGO`. |
| Inventory balance test sorted only by `event_ts`. | Updated it to match production ordering by `(event_ts, trace_id)`. |
| Generated notebooks drifted from source modules. | Rebuilt committed notebooks from templates. |
